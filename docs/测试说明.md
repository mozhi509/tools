# 单元测试说明

## 命令

| 命令 | 说明 |
|------|------|
| `npm run test` | 根目录执行：先 `test:server`，再 `test:client`（一次性、非 watch） |
| `npm run test:server` | Jest，配置见根目录 `jest.config.js`，测试目录 `server/tests/` |
| `npm run test:client` | Create React App，`client/src/**/*.test.ts(x)` |

## 后端（server/tests）

| 文件 | 内容 |
|------|------|
| `tools.routes.test.ts` | JSON format/validate/minify、Base64、URL 编解码 |
| `health.test.ts` | `GET /api/health` 行为 |
| `share.routes.test.ts` | 分享创建与读取（`redis` 已 mock） |
| `chat.routes.test.ts` | 聊天创建、加入、消息（`redis` 已 mock） |

依赖：`supertest`、`jest`、`ts-jest`。无需启动真实 Redis。

## 前端（client）

| 文件 | 内容 |
|------|------|
| `src/config/api.test.ts` | `API_ENDPOINTS` 路径、`resolveApiAssetUrl` |
| `src/components/themes.test.ts` | `getThemeColors`、主题回退 |
| `src/App.test.tsx` | 应用壳与备案链接（`fetch` mock） |
| `src/components/tools/HashCalculator.test.tsx` | 哈希页冒烟 |

### React Router 与 Jest

React Router v7 在默认 Jest 解析下易报错。项目使用 **`client/__mocks__/react-router-dom.tsx`** 作为替代实现，并在 **`client/package.json`** 的 `jest.moduleNameMapper` 中将 `^react-router-dom$` 映射到该文件。

集成测试或需要真实路由行为时，可单独配置环境或使用 Playwright 等。

## 与 CI

根目录 `npm run build` 不包含测试；建议在 CI 中增加一步 `npm run test`。
