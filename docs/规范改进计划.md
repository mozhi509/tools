# 规范改进计划

依据 `docs/client/代码规范.md`、`docs/server/代码规范.md` 及 `scripts/check-standards.js`（build 门禁）整理，按优先级分阶段执行。

---

## 现状摘要

| 维度 | 状态 |
|------|------|
| **Build 门禁** | `npm run build` 前置 `npm run check`，通过后方可编译 |

## 阶段 P0：解除 build 阻塞（必做）

**目标**：`npm run check` 通过，保证 `npm run build` 可完整跑通。

### P0-1 扩展 `client/src/config/api.ts`

在 `API_ENDPOINTS` 中补充当前缺失、但业务已使用的端点：

- **share**：`create`（POST）、`get(shareId)`（GET）
- **video**：`upload`、`trim`、`merge` 及后续若存在的 `filter`、`download`、`info`、`cleanup` 等（按 `VideoEditor` 实际调用逐项列出）

原则：所有 URL 仍通过 `API_BASE_URL` 拼接，开发与生产环境切换逻辑只保留在此文件。

### P0-2 替换硬编码 API

| 文件 | 动作 |
|------|------|
| `JsonFormatter.tsx` | `fetch('/api/tools/...')`、`fetch('/api/share/...')` → `API_ENDPOINTS.json.*`、`API_ENDPOINTS.share.*` |
| `SimpleJsonFormatter.tsx` | 同上 JSON 相关调用 |
| `VideoEditor.tsx` | 使用 `API_ENDPOINTS.video.*`；下载用 `resolveApiAssetUrl` |

### P0-3 验收

```bash
npm run check
npm run build
```

两项均成功即 P0 完成。

---

## 阶段 P1：与规范一致性与可维护性

**目标**：减少例外、统一模式，降低后续违规概率。

### P1-1 请求层统一

- **首选**：`VideoEditor` 与其它工具页统一为 `fetch` + `async/await`，错误在 `try/catch` 中处理并提示用户。
- **若保留 axios**：在 `docs/client/代码规范.md` 中增加**明确例外条款**（仅限上传 multipart 等场景），且仍必须通过 `API_ENDPOINTS` 引用地址。

### P1-2 类型与错误处理（Client）

- 将 `catch (error: any)` 改为 `unknown` + `instanceof Error` / 响应体类型收窄（与 server 规范精神一致）。
- 避免在业务组件中滥用 `any`。

### P1-3 导航与路由对齐

- **方案 A**：为「哈希」「加密」等占位工具补最小路由与占位页。
- **方案 B**：从 `ToolNavigation` 的 `toolCategories` 中移除未实现项，避免用户点到 404。

选一种并在 `docs/client/代码规范.md` 中注明「导航与 `App.tsx` 路由必须同步」。

### P1-4 自动化检查补强（可选）

在 `scripts/check-standards.js` 中逐步增加可稳定实现的规则，例如：

- client 中 `catch (error: any)` 的扫描（与 server 对齐）
- 禁止 `http://localhost:3001` 出现在 `client/src`（`api.ts` 除外）

每加一条规则需先修复存量代码再合并，避免长期红灯。

## 阶段 P2：质量与工程化

**目标**：长期可维护、CI 友好。

### P2-1 清理 ESLint 告警

针对此前 `npm run client:build` 在 `CI=true` 下失败的问题，处理：

- `ImageEditor.tsx`、`JsonFormatter.tsx`、`MarkdownEditor.tsx`、`RegexTester.tsx`、`UuidGenerator.tsx`、`clipboard.ts` 等告警项  
或  
- 在 CI 文档中说明是否使用 `DISABLE_ESLINT_PLUGIN=true`（仅作过渡，不推荐长期依赖）。

### P2-2 文档与 Skill 同步

- `docs/功能列表.md`：补充 `API_ENDPOINTS` 约定、分享与视频端点说明。
- `.cursor/skills/check-before-build`：若检查脚本规则有增删，在 skill 中同步「检查内容」小节。

### P2-3 环境变量与部署

- 新增或变更后端地址相关变量时，同步 `.env.example`（符合 `docs/server/代码规范.md`）。
