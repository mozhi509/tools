# Client 代码规范

前端（React + TypeScript）代码风格与结构约定。

---

## 一、技术栈

- **框架**: React 18
- **语言**: TypeScript
- **路由**: React Router v7
- **UI**: Ant Design 6、lucide-react 图标
- **构建**: react-scripts (Create React App)

---

## 二、目录结构

```
client/src/
├── components/          # 组件
│   ├── tools/          # 工具页组件（按功能一个文件）
│   ├── ToolNavigation.tsx
│   ├── themes.ts        # 主题色与 getThemeColors
│   └── ...
├── config/
│   └── api.ts          # API_BASE_URL、API_ENDPOINTS
├── utils/
│   └── clipboard.ts    # 剪贴板等通用工具
├── App.tsx
├── App.css
├── index.tsx
└── index.css
```

- 新增工具：在 `components/tools/` 下新增 `XxxTool.tsx`，在 `App.tsx` 中加路由，在 `ToolNavigation` 的 `toolCategories` 中加一项。
- **导航与路由必须同步**：`ToolNavigation` 中每个 `path` 必须在 `App.tsx` 的 `<Routes>` 中有对应 `<Route>`；未上线的工具不得出现在导航中（避免 404）。
- 接口地址统一在 `config/api.ts` 维护，组件内通过 `API_ENDPOINTS` 调用。

## 三、组件约定

### 3.1 组件定义

- 使用函数组件 + TypeScript。
- 类型标注：`const ComponentName: React.FC = () => { ... }` 或 `const ComponentName: React.FC<Props> = (props) => { ... }`。
- 组件文件名单词首字母大写，与组件名一致，如 `JsonFormatter.tsx`。

### 3.2 状态与类型

- 用 `useState` 管理本地状态，复杂结构先定义接口再作为泛参，例如：

```ts
interface ToolState {
  id: string;
  name: string;
  content: string;
  // ...
}
const [items, setItems] = useState<ToolState[]>([]);
```

- 避免 `any`；可选、未定结构用 `unknown` 或具体联合类型，必要时再断言。

### 3.3 工具页布局

- 工具页建议包含：
  - 顶部：`ToolNavigation`（传入 `currentTheme`）。
  - 标题区：工具名称、简要说明。
  - 主内容区：输入/输出或编辑器。
  - 页脚由 `App.tsx` 统一提供。
- 主题：从 `themes.ts` 的 `getThemeColors('vs-light')` 等获取 `ThemeColors`，用于背景、边框、按钮等，保持与导航栏一致。

### 3.4 错误处理

- `catch` 中参数类型使用 `unknown`，使用 `error instanceof Error ? error.message : String(error)` 等方式收窄后再展示或上报。

### 3.5 样式

- 优先使用 **内联 style 对象**（与现有工具页一致），便于复用 `currentTheme` 的色值。
- 全局或复用样式放在 `App.css`、`index.css`；组件专属可建 `ComponentName.css` 并 import。
- 颜色、间距等尽量来自 `ThemeColors`，避免硬编码色值。

---

## 四、API 与请求

- 所有请求基地址与端点从 `config/api.ts` 读取：
  - `API_BASE_URL`：根路径。
  - `API_ENDPOINTS.xxx`：具体接口（可为字符串或返回字符串的函数，如 `API_ENDPOINTS.chat.messages(chatId)`）。
- 使用 `fetch` 发起请求，异步用 `async/await`，错误用 `try/catch` 处理并在 UI 中提示。
- 不在此仓库内封装 axios 时，保持用 `fetch` + `Response.json()` 即可。

---

## 五、路由

- 路由在 `App.tsx` 的 `<Routes>` 中声明；工具页路径与 `ToolNavigation` 中 `path` 一致。
- 带参数路由使用 `useParams<{ paramName: string }>()` 获取参数。
- 需要跳转时使用 `useNavigate()`，避免直接改 `window.location`（除非刻意整页跳转）。

---

## 六、通用工具

- 剪贴板：使用 `utils/clipboard.ts` 的 `copyWithFeedback` 等，保证复制结果与提示一致。
- 主题：`components/themes.ts` 提供 `ThemeColors` 类型和 `getThemeColors(themeName)`，工具页统一使用以保持风格一致。

---

## 七、命名与注释

- **文件名**：PascalCase，如 `JsonFormatter.tsx`、`ToolNavigation.tsx`。
- **组件名**：与文件名一致，PascalCase。
- **变量/函数**：camelCase；常量可用 UPPER_SNAKE_CASE（如配置常量）。
- **接口/类型**：PascalCase，如 `ThemeColors`、`JsonTab`。
- 复杂逻辑、非显而易见的分支加简短中文或英文注释；对外暴露的 props 可在类型上写注释。

---

## 八、其他

- 新依赖加在 `client/package.json` 的 `dependencies` 或 `devDependencies`，版本号与现有风格一致（^ 为主）。
- 不提交调试用 `console.log`；必要时用条件判断或构建时去掉。
- 与后端约定：错误信息由后端返回 `{ error: string }` 或 `{ success: false, error: string }`，前端统一解析并展示给用户。
