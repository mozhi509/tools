## 二、前端 API 约定（`client/src/config/api.ts`）

- **`API_BASE_URL`**：生产为同源 `/api`，开发默认可为 `http://localhost:3001/api`（与后端一致）。
- **`API_ENDPOINTS`**：所有需访问后端的 URL 由此对象导出（含 `json`、`base64`、`url`、`share`、`video`、`chat` 等），业务组件**禁止**硬编码 `'/api/...'` 或带端口的完整地址（`npm run check` 会校验）。
- **静态资源 URL**：后端返回的相对路径（如 `/api/video/download/xxx`）可用 **`resolveApiAssetUrl`** 转为当前环境下可下载的绝对地址。

## 三、后端 API

### 1. 工具类 API（`/api/tools`）

- **JSON**
  - `POST /api/tools/json/format`：格式化 JSON
  - `POST /api/tools/json/validate`：验证 JSON
  - `POST /api/tools/json/minify`：压缩 JSON
- **Base64**
  - `POST /api/tools/base64/encode`：Base64 编码
  - `POST /api/tools/base64/decode`：Base64 解码
- **URL**
  - `POST /api/tools/url/encode`：URL 编码
  - `POST /api/tools/url/decode`：URL 解码

### 2. 视频 API（`/api/video`）

- `POST /api/video/upload`：上传视频（最大约 100MB）
- 视频裁剪、合并、滤镜等剪辑相关接口（依赖 ffmpeg）

### 3. 分享 API（`/api/share`）

- `POST /api/share/create`：创建分享，返回短链接（数据存 Redis，24 小时有效）
- `GET /api/share/:shareId`：根据分享 ID 获取分享数据

### 4. 聊天 API（`/api/chat`）

- `POST /api/chat/create`：创建会话，返回 `chatId` / `chatUrl`
- `POST /api/chat/:chatId/join`：加入会话
- `GET/POST /api/chat/:chatId/messages`：拉取/发送消息

### 5. 其他

- `GET /api/health`：健康检查

## 四、技术特性

- **前端**：React + TypeScript + Ant Design + React Router
- **后端**：Express，支持 CORS、Helmet、请求体大小限制、API 速率限制（如 15 分钟 100 次）
- **存储**：Redis（分享数据、会话等）
- **部署**：支持 Docker、PM2（见 `Dockerfile`、`ecosystem.config.js`）
- **规范门禁**：根目录 `npm run build` 前执行 `npm run check`（见 `scripts/check-standards.js`）
- **单元测试**：根目录 `npm run test`（后端 `server/tests` + 前端 `client/src/**/*.test.*`），详见 [测试说明](./测试说明.md)

## 五、说明

- 分享链接在生产环境依赖 `DOMAIN`、`HTTPS_ENABLED` 等配置生成正确 URL。
- 哈希/加密工具为浏览器端计算；加密页为演示用途，重要数据请使用专业方案。
